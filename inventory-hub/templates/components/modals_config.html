🔢 Restore Spoolman Field Order

Spoolman doesn't preserve a canonical sort order on its extra fields — every field drifts to order: 0 after edits, and Spoolman's own UI shows them in mutation order. This writes FCC's FIELD_ORDER canonical index back to each field so Spoolman renders extras in the same sequence as the wizard / details modal.

Safety: Preview is a dry-run — it shows the exact list of moves without writing anything. Only the Apply button on the preview commits changes. Each write echoes Spoolman's full field definition (name, type, choices, etc.) back to it, so the only property that ever changes is order. Idempotent — fields already in position skip the round-trip; keys not in FCC's FIELD_ORDER pass through untouched.

Click Preview changes to start.
